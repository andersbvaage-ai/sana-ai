Hvert krav er implementert arkitekturelt — ikke bare dokumentert i en policy.

GDPR Art. 9 — særlige kategorier
Personvernforordningen
Helseopplysninger er særlige kategorier persondata under GDPR Art. 9 og krever eksplisitt rettslig grunnlag og tilleggssikring. Sana AI behandler aldri helseopplysninger uten at PII er fjernet lokalt først.
Tiltak: PII-scrubbing skjer på norsk server (Elastic Beanstalk eu-north-1) før data forlater systemet. AI-modellen mottar aldri navn, fødselsnummer eller kontaktdata — kun klinisk innhold.
Normen — norm for informasjonssikkerhet
Helsesektornormen
Normen er bransjestandarden for informasjonssikkerhet i helse- og omsorgssektoren i Norge. Den stiller krav til tilgangsstyring, logging og hendelseshåndtering.
Tiltak: Rollebasert tilgangskontroll (RBAC), JWT-sesjon med 8 timers utløp, bcrypt-passordhashing, og fullstendig logging av alle tilgangsoperasjoner til CloudWatch.
Pasientjournalloven
Lov om pasientjournal
Pasientjournalloven regulerer hvordan helseopplysninger i journaler kan behandles, hvem som har tilgang, og krav til integritet og sporbarhet.
Tiltak: Uforanderlig revisjonslogg for alle operasjoner på klagesaker. Kun autentiserte brukere med riktig rolle kan lese eller oppdatere en sak. Sletting er ikke mulig via UI.
AWS eu-north-1 — dataresidency
Geografisk datakontroll
All behandling og lagring skjer i AWS eu-north-1 (Stockholm, Sverige) — innenfor EØS. Data overføres ikke til tredjeland. AWS Bedrock-kall er konfigurert til eu-north-1 endpoint.
Tiltak: Explicit region-parameter på alle AWS SDK-kall. S3-bøtter er konfigurert med region-lock. Ingen CDN-distribusjon av sensitiv data utenfor EØS.
JWT-autentisering og tilgangskontroll
Identitet og sesjonsstyring
Alle API-endepunkter krever gyldig JWT-token. Tokens har 8 timers levetid og bruker HS256-signering. Passord lagres som bcrypt-hash med tilstrekkelig kostfaktor.
Tiltak: requireAuth-middleware på alle beskyttede ruter. Brukerliste styres via miljøvariabel, ikke hardkodet. Sesjon invalideres ved utlogging.
Uforanderlig revisjonslogg
Audit trail
Hvert steg i klagesaksbehandlingen — opplasting, PII-scrubbing, AI-analyse, legesgodkjenning, eksport — loggføres med tidsstempel, bruker-ID og AI-modellversjon.
Tiltak: Logger sendes til AWS CloudWatch Logs (AUDIT_BACKEND=cloudwatch). Logger kan ikke endres eller slettes via applikasjonen. Tilgjengelig for internrevisjon og Datatilsynet.
Dataflyt

Ingen PII når AI-modellen

Det tekniske prinsippet er enkelt: personopplysninger og klinisk innhold skilles allerede på serveren. AI-modellen ser aldri hvem pasienten er.

Dette er ikke en policy som kan omgås — det er arkitekturen som håndhever det.
